<section name="fac3" class="section section--body section--first"><div class="section-divider"><hr class="section-divider"></div><div class="section-content"><div class="section-inner sectionLayout--insetColumn"><h3 name="c736" id="c736" class="graf graf--h3 graf--leading graf--title">How many data do we need to train machine hearing Mandarin Chinese?</h3><blockquote name="6c5b" id="6c5b" class="graf graf--blockquote graf-after--h3">Report from my discussion with western engineer about how many sentences and voices data we need to train a NN model to convert Mandarin voice into Chinese sentence, explain why I don’t thinnk we need our sample contains a million different sentences.</blockquote><p name="d6a8" id="d6a8" class="graf graf--p graf-after--blockquote graf--trailing">(update: re-write for clearly explain on 26 Feb 3:40 UTC)</p></div></div></section><section name="768d" class="section section--body"><div class="section-divider"><hr class="section-divider"></div><div class="section-content"><div class="section-inner sectionLayout--insetColumn"><p name="aae9" id="aae9" class="graf graf--p graf--leading">The amounts of voice “unit” we need in Mandarin and in English may not be that big that you think. Let me try to explain why there is that big difference in my imagination.</p><h3 name="6a1f" id="6a1f" class="graf graf--h3 graf-after--p">Tl;dr, The final Deep Speech model we need is actually a Voice-Phonetics model, rather than a Voice-Chinese model.</h3><p name="bb6c" id="bb6c" class="graf graf--p graf-after--h3">When machine hears the voice, its output should not be “Chinese characters”, but “romanized phonetics” (either in Zhuyin or Pinyin).</p><p name="9ca3" id="9ca3" class="graf graf--p graf-after--p">We feed the model with _best_browser_in_the_world.mp3_,<br>and the output should be,</p><blockquote name="47dc" id="47dc" class="graf graf--blockquote graf-after--p">- “g4 ru,4 g;4 yjo4 cl3 2k7 xu.6 x03 fu4” (in Zhuyin)<br>- “Shì jiè shàng zuì hǎo de liú lǎn qì” (or in Pinyin)</blockquote><p name="8048" id="8048" class="graf graf--p graf-after--blockquote">but not,</p><blockquote name="4405" id="4405" class="graf graf--blockquote graf-after--p">世界上最好的瀏覽器 (in Chinese characters)</blockquote><p name="a045" id="a045" class="graf graf--p graf-after--blockquote">The phonetic-based input method (which all of us use them) is a classic and popular open source topic been researched for 20 years,</p><p name="c704" id="c704" class="graf graf--p graf-after--p">Once people got the above output, we can easily convert it back to “世界上最好的瀏覽器”, with Zhuyin or Pinyin dictionary table*, which contains all phonetics combination of phrases/chars and it’s frequency.</p><p name="f112" id="f112" class="graf graf--p graf-after--p graf--trailing">*<a href="https://sites.google.com/site/ianho7979/InputMethodTables" data-href="https://sites.google.com/site/ianho7979/InputMethodTables" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Zhuyin table</a> for example</p></div></div></section><section name="c06f" class="section section--body"><div class="section-divider"><hr class="section-divider"></div><div class="section-content"><div class="section-inner sectionLayout--insetColumn"><h3 name="fe13" id="fe13" class="graf graf--h3 graf--leading">Why we want the model to output pronunciation but not Chinese characters?</h3><p name="4906" id="4906" class="graf graf--p graf-after--h3">Because we have too many characters.</p><p name="9e4d" id="9e4d" class="graf graf--p graf-after--p">Consider 5k common characters (which most people only learn and use this amount, ideally, if we can have samples on all the sentences that people would speak, so it can learn to identify every sentence with their voice combination. with 20 chars length sentences, that’s 5k²⁰ combination, so that’s not gonna work.</p><p name="f0a2" id="f0a2" class="graf graf--p graf-after--p">So, in practice, we break the sentences into the phrase (2~3 syllables) and single char (one syllable). Many database recording voice in phrase but not sentences, and the model just needs to find the phases/chars for everyone, two, or three syllables. In Chinese, we only have 5k³+5k*2+5k combination, that’s 125000M, much realistic.</p><h4 name="4b11" id="4b11" class="graf graf--h4 graf-after--p">A voice-phonetics model</h4><p name="a6cb" id="a6cb" class="graf graf--p graf-after--h4">Actually, we don’t need that much sample too. Because those 5k chars only share the same 1500 pronunciation. if we ask our model not to identify different chars, but different phrase pronunciations, we can easily reduce the combination of samples down to 1.5k³+1.5k*2+1.5k = 3375M.</p><p name="a85a" id="a85a" class="graf graf--p graf-after--p graf--trailing">How can we further reduce the samples we need? We break the sentences into character, each character only had one syllable, that’s our advantage on Mandarin have — the model should be easily split clip by looking at the pause and level of the input voice. So I believe the minimum we only need 1.5k samples, in fact.</p></div></div></section><section name="7a95" class="section section--body"><div class="section-divider"><hr class="section-divider"></div><div class="section-content"><div class="section-inner sectionLayout--insetColumn"><h3 name="3917" id="3917" class="graf graf--h3 graf--leading">Example</h3><p name="28cc" id="28cc" class="graf graf--p graf-after--h3">Take this sentence as an example:</p><blockquote name="1fb0" id="1fb0" class="graf graf--blockquote graf-after--p">世界上最好的瀏覽器 (The best browser in the world)</blockquote><p name="5180" id="5180" class="graf graf--p graf-after--blockquote">If we want Our machine to listent to the whole sentences, and find it’s identity characters, that is -</p><blockquote name="1f03" id="1f03" class="graf graf--blockquote graf-after--p">世 界 上 最 好 的 瀏 覽 器</blockquote><p name="caa9" id="caa9" class="graf graf--p graf-after--blockquote">We need the machine to learn all the combination of all the characters, that’s when we need 5k²⁰ samples. Not gonna work.</p><p name="42fa" id="42fa" class="graf graf--p graf-after--p">How can we reduce it?</p><h4 name="afdb" id="afdb" class="graf graf--h4 graf-after--p">Phrase based voice-phonetics model</h4><p name="a5b7" id="a5b7" class="graf graf--p graf-after--h4">We train machines to identify the pronunciation of phrases, but not Chinese characters. We already know Mandarin had around 1500 different pronunciation. so if we consider only 2 char length phrases, our model only need to output this result,</p><blockquote name="ec1f" id="ec1f" class="graf graf--blockquote graf-after--p">- Shìjiè shàng zuì hǎo de liúlǎn qì (Hanyu Pinyin, you can get this by copy-paste the whole sentences into Google Translate)<br>- ㄕˋㄐㄧㄝˋ / ㄕㄤˋ / ㄗㄨㄟˋㄏㄠˇ / ㄉㄜ˙ / ㄌㄧㄡˊㄌㄢˇ / ㄑㄧˋ (Zhuyin)<br>- g4ur,4 / g;4 / yjo4cl3 / 2k7 / xu.6x03 / fu4 (Zhuyin rep. by keyboard char)</blockquote><p name="5b2a" id="5b2a" class="graf graf--p graf-after--blockquote">People who had this output can use pronunciation-chars dictionary table, another model, or other methodology to figure out which words are using in the sentences, and have the final result,</p><blockquote name="47be" id="47be" class="graf graf--blockquote graf-after--p">世界 / 上 / 最 / 好 / 的 / 瀏覽 / 器</blockquote><p name="0946" id="0946" class="graf graf--p graf-after--blockquote">The problem now looks much much similar to English and other roman-character based languages. We only need 1.5k*1.5k samples, that’s 225M samples.</p><h4 name="7a13" id="7a13" class="graf graf--h4 graf-after--p">Character based voice-phonetics model</h4><p name="015a" id="015a" class="graf graf--p graf-after--h4">We can even reduce the data we need, because we actually only need our model to identity chars, but not phrases. The output we really need is,</p><blockquote name="3685" id="3685" class="graf graf--blockquote graf-after--p">-Shì jiè shàng zuì hǎo de liú lǎn qì (Hanyu Pinyin)<br>-ㄕˋ / ㄐㄧㄝˋ / ㄕㄤˋ / ㄗㄨㄟˋ / ㄏㄠˇ / ㄉㄜ˙ / ㄌㄧㄡˊ / ㄌㄢˇ / ㄑㄧˋ (Zhuyin)<br>- g4 / ru,4 / g;4 / yjo4 / cl3 / 2k7 / xu.6 / x03 / fu4 (Zhuyin keyboard char)</blockquote><p name="7300" id="7300" class="graf graf--p graf-after--blockquote">And the same, people can easily convert the output back to the final sentences,</p><blockquote name="9e5d" id="9e5d" class="graf graf--blockquote graf-after--p graf--trailing">世 界 上 最 好 的 瀏 覽 器</blockquote></div></div></section><section name="df90" class="section section--body"><div class="section-divider"><hr class="section-divider"></div><div class="section-content"><div class="section-inner sectionLayout--insetColumn"><p name="5f90" id="5f90" class="graf graf--p graf--leading">Although we have 5k chars, we only have 1500 sounds, that can represent by 26 characters, each with 4 tones* (if we based on Pinyin), or 37 chars with 4 tones* (base on Zhuyin) to learn. That’s much more similar to English, right?</p><p name="7bbf" id="7bbf" class="graf graf--p graf-after--p">* some researchers told me that we don’t even need tones in this scenario.</p><p name="5ebf" id="5ebf" class="graf graf--p graf-after--p graf--trailing">That is also why we can still use 36-key English keyboard to type Chinese, but not a keyboard with a few hundred keys — which really exist in our computer history.</p></div></div></section><section name="6092" class="section section--body"><div class="section-divider"><hr class="section-divider"></div><div class="section-content"><div class="section-inner sectionLayout--insetColumn"><h3 name="9253" id="9253" class="graf graf--h3 graf--leading">How do we train our model?</h3><p name="8a87" id="8a87" class="graf graf--p graf-after--h3">In practice, how can we train the model with this pronunciation-based strategy?</p><p name="4c6c" id="4c6c" class="graf graf--p graf-after--p">We don’t feed the model with the original Chinese sentences. We should convert the sentence into Pinyin / Zhuyin and feed it.</p><blockquote name="260f" id="260f" class="graf graf--blockquote graf-after--p graf--trailing">You feed best_browser_in_the_world.mp3<br>with <br>- “g4 ru,4 g;4 yjo4 cl3 2k7 xu.6 x03 fu4” (convert by Zhuyin table)<br>- “Shì jiè shàng zuì hǎo de liú lǎn qì” (convert by Pinyin table)</blockquote></div></div></section><section name="ee2c" class="section section--body"><div class="section-divider"><hr class="section-divider"></div><div class="section-content"><div class="section-inner sectionLayout--insetColumn"><p name="d674" id="d674" class="graf graf--p graf--leading graf--trailing">I find that this discuss may be useful for others, so I re-published it from <a href="https://discourse.mozilla.org/t/number-of-sentences-needed/36182" data-href="https://discourse.mozilla.org/t/number-of-sentences-needed/36182" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Mozilla Discourse</a>.</p></div></div></section><section name="a236" class="section section--body section--last"><div class="section-divider"><hr class="section-divider"></div><div class="section-content"><div class="section-inner sectionLayout--insetColumn"><figure name="3c99" id="3c99" class="graf graf--figure graf--iframe graf--leading graf--trailing"><iframe src="https://button.like.co/in/embed/irvinchen/button/" width="485" height="212" frameborder="0" scrolling="no"></iframe></figure></div></div></section>
